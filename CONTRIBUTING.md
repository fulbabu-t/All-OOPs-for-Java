# 🤝 CONTRIBUTING.md

Thank you for your interest in contributing to this Java OOP Concepts project! 🎉  
Your contributions help improve learning resources for developers and students exploring Object-Oriented Programming in Java.

---

# 📌 Contribution Guidelines

Please follow these guidelines before contributing to the repository.

---

# 🚀 How to Contribute

## 1️⃣ Fork the Repository

Click the **Fork** button at the top-right corner of the repository page.

---

## 2️⃣ Clone Your Fork

```bash
git clone https://github.com/your-username/repository-name.git
```

Move into the project directory:

```bash
cd repository-name
```

---

## 3️⃣ Create a New Branch

Create a branch for your feature or fix.

```bash
git checkout -b feature-name
```

Example:

```bash
git checkout -b add-java-source-files
```

---

## 4️⃣ Make Your Changes

You can contribute by:

- Adding `.java` source files
- Improving documentation
- Adding comments
- Creating new OOP examples
- Fixing bugs
- Organizing project structure
- Adding unit tests

---

## 5️⃣ Commit Your Changes

```bash
git add .
git commit -m "Added new OOP example"
```

---

## 6️⃣ Push to GitHub

```bash
git push origin feature-name
```

---

## 7️⃣ Create a Pull Request

Open a Pull Request (PR) from your forked repository to the main repository.

Please provide:
- Clear title
- Proper description
- Screenshots (if applicable)

---

## Intermediate Contributions

- Add package structure

- Add collections framework examples

## Advanced Contributions

- Add Maven/Gradle support
- Add JUnit testing
- Improve architecture
- Add design pattern examples

# 🛠 Coding Standards

Please follow these coding standards:

- Use meaningful class and method names
- Write clean and readable code
- Add comments where necessary
- Follow standard Java naming conventions
- Avoid unnecessary code duplication

---

# ✅ Pull Request Checklist

Before submitting your PR, ensure that:

- [ ] Code compiles successfully
- [ ] No unnecessary files are added
- [ ] README is updated if required
- [ ] Changes are properly tested
- [ ] Code follows project structure

---

# 🐞 Reporting Issues

If you find bugs or want to suggest improvements:

1. Open the **Issues** tab
2. Create a new issue
3. Provide detailed information

Include:
- Problem description
- Expected behavior
- Screenshots (if available)
- Suggested solution
